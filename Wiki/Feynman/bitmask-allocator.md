# Bitmask-аллокатор + сводка дырок по регионам — «методом Фейнмана»

## Проблема

Мы пишем тела блоков в сегменты **в конец** (append-only). Когда блок удаляют — внутри сегмента
остаётся **дырка**. Чтобы вернуть это место, обычно приходится **переписать весь сегмент** (компакция)
— дорого. Можно ли переиспользовать дырки **точечно**, не переписывая всё?

## Аналогия — гардероб с номерками и доской «где свободно»

Большой гардероб с крючками. Чтобы быстро находить место для пальто, заводим систему.

1. ☑️ **Доска занятости (битмаска).** На каждый крючок — **одна лампочка**: горит = занят, погасла =
   свободен. Удалили пальто — лампочка гаснет, крючок снова доступен (без перевешивания всего ряда).
2. 🗺️ **Сводка по секциям (region-gaps).** Гардероб разбит на **секции**. На каждой секции — табличка:
   «самый длинный ряд свободных крючков = N», плюс «свободно с краёв». Не надо обходить все лампочки.
3. 🎯 **Быстрый поиск места.** Нужно повесить вещь на 3 крючка подряд? Смотрим **таблички секций**,
   находим секцию, где «макс. ряд ≥ 3», и сканируем **только её** — не весь гардероб.
4. 🧵 **Сшиваем на границах.** «Свободно с конца секции A» + «свободно с начала секции B» = длинный ряд
   через границу. Поэтому храним `leading/trailing`, а не только `max`.
5. ♻️ **Точечный re-use без уборки.** Дырки переиспользуются на лету → **полная компакция не нужна**
   (или нужна реже).

> Чертёж — из Qdrant `gridstore` (Rust!): `Bitmask` (1 бит/блок) + `RegionGaps{max, leading, trailing}`.
> Это реальная Rust-реализация, её код можно взять за образец. Родственно
> [disk-allocator.md](disk-allocator.md) (Dragonfly segmented) и [fixed-block-allocator.md](fixed-block-allocator.md).

## Проверка понимания

> Свободное место — **битмаска по блокам** (бит = занят/свободен), а поверх — **сводка на регион**
> (длиннейший свободный прогон + края). Место под N блоков ищем **по сводкам секций, не сканируя всё**,
> и переиспользуем дырки **точечно, без полной компакции**.

## Зачем это нам

Фаза 1/5: четвёртый вариант `segment_alloc` (рядом с append-only / fixed-block / segmented) —
вернуть место от удалённых блоков **без переписывания сегментов**, с быстрым поиском места.
