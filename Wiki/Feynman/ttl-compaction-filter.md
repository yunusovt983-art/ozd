# TTL через compaction-filter — «методом Фейнмана»

## Проблема

Часть блоков **временные** (ephemeral, с TTL — истекают). Если они **сгруппированы по времени** в
один сегмент — удаляем [сегмент-окно целиком](#) по возрасту, дёшево. Но если временные блоки
**перемешаны** с долгоживущими в общих сегментах — как выкидывать истёкшие, **не запуская отдельный
дорогой проход** «прочитать всё → найти просроченное → удалить»?

## Аналогия — выбрасывать просрочку во время плановой переборки холодильника

Не устраиваешь спец-рейд «найти просрочку» — просто, **когда уже перебираешь** холодильник (плановая
уборка), заодно **выкидываешь просроченное**:

1. 🧹 **Совместить с компакцией.** GC всё равно периодически [перепаковывает](splice-merge.md) живые
   блоки в новый сегмент. **На этом же проходе** проверяем каждый блок.
2. 🗓️ **Фильтр по embedded-ts.** У ephemeral-блока в метаданных — отметка времени. Истёк
   (`ts + ttl < now`)? → **не переносим** в новый сегмент (выпал). Жив? → переносим.
3. 🚫 **Без отдельного скана.** Никакого дополнительного полного прохода по диску ради TTL — всё
   делается **попутно** с уже идущей компакцией. Бесплатно по I/O.

> Чертёж из Flink (`RocksDbTtlCompactFiltersManager`: `FlinkCompactionFilter` отбрасывает истёкшие
> записи на проходе компакции RocksDB). Дополняет [time-bucketed drop целого окна](#) (когда блоки
> сгруппированы) и FIFO=TTL.

## Проверка понимания

> **TTL через compaction-filter** = выкидывать истёкшие блоки **попутно на проходе компакции/GC** (по
> embedded-ts, «не переносить в новый сегмент»), без отдельного дорогого скана — для перемешанных TTL,
> которые нельзя сгруппировать в отдельное окно.
