# HRW-placement (Rendezvous hashing) — объяснение «методом Фейнмана»

## Какой вопрос решаем

У нас 60 дисков. Для **каждого** блока надо решить: **на каком диске (дисках) он лежит** — причём
так, чтобы:
1. это считалось **по формуле** (без центрального списка «что где»);
2. при **добавлении/удалении диска** переезжало **как можно меньше** блоков.

## Почему «номер % 60» — плохо

Наивно: «диск = номер_блока **mod** 60». Работает… пока число дисков не меняется. Добавили
**61-й диск** — формула стала `mod 61`, и **почти каждый блок** теперь должен переехать на другой
диск. Это как если бы в гардеробе **перенумеровали все вешалки** из-за одной новой — всем пришлось
бы перевесить пальто. Катастрофа на 480 ТБ.

## Аналогия — лотерея между блоком и дисками

**HRW (Highest Random Weight)** делает иначе:
- Для пары **(блок X, диск D)** считаем «лотерейный билет» = `hash(X, D)` — число.
- Блок едет на тот диск, у которого **самый крупный билет** для этого блока. Для **R копий** —
  на R дисков с самыми крупными билетами (top-R).

**Что происходит при удалении диска?** Уходит только **его** «выигрышный» билет. Блоки, которые
выигрывал ушедший диск, перевыбирают **следующего по величине билета**. **Все остальные блоки не
двигаются** — их победитель на месте. Переезжает лишь доля ≈ `1/число_дисков`, а не «почти всё».

Это как лотерея, где у каждого пальто **свой** список «кому достанусь, если эта вешалка занята» —
убрали одну вешалку, перевесили только пальто с неё, остальные висят как висели.

**Бонус:** билеты можно **взвесить по свободному месту** — диск посвободнее «выигрывает» чаще,
нагрузка балансируется сама.

## Проверка понимания

> **HRW** для каждого блока разыгрывает «лотерею» между дисками (`hash(блок, диск)`), и блок едет к
> диску(ам) с наибольшим числом. Так расположение **считается формулой** (без общего списка), а при
> добавлении/удалении диска переезжает лишь крошечная доля блоков, а не почти все, как при «mod N».

## Зачем это нам

HRW даёт нам **детерминированное размещение без центрального каталога** (его источник правды —
сама формула) и **дешёвый resilver** (переезжает только дельта). top-R по HRW = выбор реплик.
