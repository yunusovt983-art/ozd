# SeqLock: lock-free чтение горячего состояния — «методом Фейнмана»

## Проблема

Некоторые данные читают **очень часто и многие сразу**: веса свободного места дисков, статистика кэша,
горячий хвост индекса. Меняются они **редко**. Если на каждое чтение брать обычный лок — десятки
воркеров **дерутся за него** впустую (хотя писатель почти не приходит).

## Аналогия — табло вылетов на вокзале

Сотни людей **смотрят** на табло вылетов одновременно. Никто не «берёт табло в руки» — все просто
читают. Изредка диспетчер **обновляет** его.

1. 🔢 **Счётчик версии.** У табло есть **номер версии**. Чётный = «стабильно, читать можно». Нечётный =
   «диспетчер сейчас меняет, погоди».
2. 👀 **Читатель не блокирует.** Читатель: запомнил версию → прочитал табло → снова посмотрел версию.
   **Совпала** (и была чётной) → прочитал консистентно. **Не совпала** → диспетчер влез во время
   чтения → просто **перечитать** (ретрай). Читатель **никого не ждёт на локе**.
3. ✍️ **Писатель помечает «идёт правка».** Диспетчер: версия +1 (стала нечётной = «правлю») →
   поменял → версия +1 (снова чётная = «готово»).
4. ⚡ **Почему это быстро.** Чтений много, записей мало → читатели почти всегда проходят с первого раза,
   **без contention**. Идеально для горячего, редко-меняющегося состояния.
5. 🚧 **Где НЕ применять.** Не для самих данных блоков (они большие, читаются один раз) — только для
   маленького горячего состояния (веса, счётчики, snapshot топологии).

> Чертёж — из Qdrant `trififo::seqlock` (Rust). Сочетается с [entity-actor.md](entity-actor.md)
> (сериализация по CID) и [read-coalescing.md](read-coalescing.md).

## Проверка понимания

> **SeqLock** = счётчик версии: читатель читает без лока и проверяет, что версия не изменилась (иначе
> ретрай), писатель помечает «идёт правка» нечётной версией. Для **горячего, редко-меняющегося**
> состояния (веса free-space, статы кэша) — много читателей не дерутся за лок.

## Зачем это нам

Фаза 2: на 60-wide параллелизме читать веса свободного места и snapshot топологии для placement
**без блокировок читателей** — меньше contention на горячем пути.
