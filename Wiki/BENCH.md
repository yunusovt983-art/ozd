# ozd-bench — результаты прогонов

Харнесс: `cargo run -p ozd-bench --release -- --disks=6 --objects=600 --reads=4000 …`
(in-process storage-слой, без HTTP; Kubo-микс тел 75%×256КиБ / 15%×4КБ / 10%×1МиБ;
hot-set 10% ключей / 90% трафика; фиксированный seed=42 — отчёт воспроизводим).

## Прогон 2026-06-11 — dev-машина (macOS, внешний SSD; tmpdir)

⚠️ **Оговорка**: на dev-машине «диски» — каталоги одного SSD, чтения по сути из
page cache (~7.5 ГиБ/с) — это потолок software-пути, НЕ полка. Сравнивать
строки между собой можно, абсолютные цифры HDD-полки даст стенд E30/E32.

| Профиль | PUT p50/p99 | GET p50/p99 | GET ops/s | hit-rate | прочее |
|---|---|---|---|---|---|
| mirror, без кэша | 13.8/20.6мс | 0.10/0.48мс | 30139 | — | hedge_thr=10мс (адаптив у пола) |
| mirror + СуперДиск 512М | 17.9/26.8мс | 0.08/0.77мс | 28774 | **100%** | coalesced=153, evicted=0 |
| erasure 4+2 + СуперДиск | 22.8/37.9мс | 0.08/0.59мс | 35490 | **100%** | put = +RS-кодирование |

Наблюдения:
- **PUT**: зеркало пишет 2 полные копии; EC 4+2 — 6 кусков (1.5× байт) + RS-математика
  → p50 22.8мс против 13.8мс. На HDD-полке разрыв сожмётся (диски параллельные).
- **GET через пул без кэша**: mirror p50 0.07мс vs **EC p50 0.40мс** (сборка 4 кусков
  с 4 «дисков») — ×5.7. СуперДиск этот разрыв прячет полностью (0.08мс в обоих) —
  подтверждение «синергии с EC» из E25.
- **Write-through** даёт 100% hit-rate, пока рабочий сет (≈230МиБ) < бюджета кэша;
  реальную кривую hit-rate vs размер кэша снимет стенд.
- Coalescing срабатывает даже в бенче (4 потока × hot-set): 153–164 слитых чтения.

## Вердикт E24 (микроблоки 16КиБ #15) — данными

Текущий путь читает 256КиБ-тело целиком ради 16КиБ диапазона (амплификация 16×),
verify слайса поверх — копейки (0.02мс). **НО**: Kubo (go-ds-s3) читает блоки
ТОЛЬКО целиком — Range-трафика в Kubo-профиле не существует. Микроблоки добавили
бы 16× записей в индекс и усложнили GC ради нагрузки, которой нет.

**Решение: E24 остаётся 🧊.** Триггер пересмотра: появление прямого S3-Range
трафика в профиле стенда (метрики 206-ответов шлюза).

## Как повторить

```bash
cargo run -p ozd-bench --release -- --disks=6 --objects=600 --reads=4000 \
  --redundancy=erasure --cache-mb=512 --threads=4 --seed=42
```
Ключи: `--disks --objects --reads --hot-keys --hot-traffic --redundancy=mirror|erasure
--cache-mb (0=off) --threads --seed`.
