# Автономный режим — OpenZFS Daemon

## Правило автономности

Работай автономно. **Не спрашивай подтверждений и уточнений.** Если решение неоднозначно — выбирай вариант, наиболее согласованный с `Wiki/ARCHITECTURE.md` и философией DDD (домен без IO, порты и адаптеры, инварианты агрегата). Фиксируй выбор в сообщении коммита и продолжай.

## Гейт: тесты

- НЕ коммить, если `cargo test` красный.
- После 3 неудачных попыток на задаче — пропусти её, залогируй причину в `Wiki/BLOCKED.md` и переходи к следующей.
- Закончив план арок — распланируй следующий блок задач и продолжай.

## Порядок работы

1. Читай `Wiki/WEEKLY-ARCS.md` (текущие арки).
2. Выполняй задачи по порядку: код → `cargo test` → коммит → push.
3. Коммит по каждой задаче (малые проверяемые единицы).
4. Сообщения коммитов — по-русски, краткие, с номером арки (напр. `W1.1: degraded start при ошибке открытия шарда`).

## Стандарты кода

- Идентификаторы — английские; комментарии к коду — по-русски.
- DDD: домен (`ozd-domain`) без IO; адаптеры зависят от домена.
- Ноль паник на горячем пути; `unwrap()` только в тестах.
- Существующие паттерны проекта важнее «лучших практик из интернета».
- Не добавлять зависимости без крайней необходимости.

## Команды проверки

```
cargo test
cargo build --release
cargo clippy -- -D warnings
```

## Ветка

Ночные автономные прогоны — на ветке `feature/night` (не main). Merge в main — только после утреннего ревью.
